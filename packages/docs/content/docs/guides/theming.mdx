## Overview

Polyterm UI includes a React context-based theming system with semantic color tokens. All components read from the theme by default, and explicit color props always override theme values.

<Callout type="info">
  Without a `ThemeProvider`, all components fall back to the built-in dark theme — existing code works unchanged.
</Callout>

## Live Demo

Use **←→** arrow keys to switch between dark and light themes. All components update automatically.

## Quick Start

```tsx title="app.tsx"
import { ThemeProvider, darkTheme, Spinner, Table } from "@polyterm.io/ui"

function App() {
  return (
    <ThemeProvider theme={darkTheme}>
      <Spinner text="Loading..." />
      <Table data={[{ name: "Alice", role: "Admin" }]} />
    </ThemeProvider>
  )
}
```

## Built-in Themes

Two themes ship out of the box:

```tsx
import { darkTheme, lightTheme } from "@polyterm.io/ui"
```

| Token | Dark | Light |
|-------|------|-------|
| `primary` | `"cyan"` | `"magenta"` |
| `accent` | `"cyan"` | `"magenta"` |
| `secondary` | `"green"` | `"cyan"` |
| `muted` | `"gray"` | `"gray"` |
| `border` | `"gray"` | `"gray"` |
| `text` | `"white"` | `"white"` |
| `success` | `"green"` | `"green"` |
| `error` | `"red"` | `"red"` |
| `warning` | `"yellow"` | `"yellow"` |

## Custom Themes

Create a custom theme by providing an object that satisfies the `Theme` interface:

```tsx title="my-theme.ts"
import type { Theme } from "@polyterm.io/ui"

export const myTheme: Theme = {
  primary: "blue",
  accent: "blue",
  secondary: "yellow",
  muted: "gray",
  border: "gray",
  text: "white",
  success: "green",
  error: "red",
  warning: "yellow",
}
```

Or extend a built-in theme:

```tsx title="custom-theme.ts"
import { darkTheme } from "@polyterm.io/ui"

export const customTheme = {
  ...darkTheme,
  primary: "blue",
  accent: "blue",
}
```

## Using the Theme Hook

Access theme tokens directly in your own components:

```tsx
import { useTheme } from "@polyterm.io/ui"

function MyComponent() {
  const theme = useTheme()

  return (
    <text style={{ fg: theme.primary }}>
      Themed text
    </text>
  )
}
```

## Token Reference

| Token | Purpose | Components |
|-------|---------|------------|
| `primary` | Headers, highlights, assistant messages | Table, ChatPanel, MultiSelect |
| `accent` | Active/focused interactive elements | TabBar, TextInput, SelectInput |
| `secondary` | User messages, checkboxes, prompts | ChatPanel, MultiSelect |
| `muted` | Spinners, placeholders, pending states | Spinner, ChatPanel (tool calls) |
| `border` | Table and modal borders | Table, Modal |
| `text` | Default foreground text | SelectInput |
| `success` | Completed states | ChatPanel (tool calls) |
| `error` | Failed states | ChatPanel (tool calls) |
| `warning` | In-progress states | ChatPanel (tool calls) |

## Props Override Theme

Explicit color props always take precedence over the theme. This lets you override individual components without creating a new theme:

```tsx
<ThemeProvider theme={darkTheme}>
  {/* Uses theme.muted ("gray") */}
  <Spinner text="Default" />

  {/* Overrides to "red" regardless of theme */}
  <Spinner text="Custom" color="red" />
</ThemeProvider>
```
