Gridland renders terminal-style UIs directly to an HTML5 Canvas using React.
Built on the [opentui](https://opentui.com) engine. No xterm.js, no terminal emulator, just a React reconciler, Yoga layout, and canvas pixels.

## Features

- **Canvas rendering** - TUI content drawn directly to `<canvas>`, not a DOM tree
- **React reconciler** - Use JSX with `<box>`, `<text>`, and other TUI primitives
- **SSR-safe** - Single `<TUI>` component with built-in client detection
- **Yoga layout** - Flexbox positioning via `yoga-layout`
- **Framework support** - Works with Vite and Next.js
- **Interaction** - Text selection, copy/paste, keyboard events, and drag-and-drop

## Quick Start

Install the package in your project:

<Tabs items={['bun', 'npm', 'yarn', 'pnpm']}>
<Tab value="bun">
```bash title="Terminal"
bun add @gridland/web
```
</Tab>
<Tab value="npm">
```bash title="Terminal"
npm install @gridland/web
```
</Tab>
<Tab value="yarn">
```bash title="Terminal"
yarn add @gridland/web
```
</Tab>
<Tab value="pnpm">
```bash title="Terminal"
pnpm add @gridland/web
```
</Tab>
</Tabs>

Then wrap your TUI content in `TUI`:

```tsx title="App.tsx"
import { TUI } from "@gridland/web"

function App() {
  return (
    <TUI style={{ width: "100vw", height: "100vh" }}>
      <box border borderStyle="rounded" padding={1}>
        <text fg="#a3be8c">Hello from Gridland!</text>
      </box>
    </TUI>
  )
}
```

<Callout type="info">
You'll also need to configure your bundler (Vite or webpack) to resolve the underlying opentui engine modules. See the framework guides below.
</Callout>
